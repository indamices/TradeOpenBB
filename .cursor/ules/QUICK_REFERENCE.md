# 🎯 代码审查快速参考
# Code Review Quick Reference - 每次任务前查看

## ⚡ 3秒检查

```bash
# 立即运行
./scripts/quick-scan.sh
```

---

## 📋 5秒确认

在开始审查前，确认：

- [ ] 我会先扫描，后理解
- [ ] 我会发现实际问题，不创造理论问题
- [ ] 我会提供具体修复代码
- [ ] 我会验证修复效果
- [ ] 我不会跳过任何步骤

---

## 🔍 必检项目（15秒快速扫描）

### Python (5秒)
```bash
grep -r "print(\|except.*: pass" . --include="*.py"
```

### TypeScript (5秒)
```bash
grep -r "console\\.log\|as any" . --include="*.tsx"
```

### 硬编码路径 (5秒)
```bash
grep -r "c:\\\\Users\|127\\.0\\.0\\.1" . --include="*.py" --include="*.tsx"
```

---

## 🚫 3个禁止

1. ❌ **不跳过扫描** - 必须运行工具扫描
2. ❌ **不看文档看代码** - 必须验证问题存在
3. ❌ **不提供模糊建议** - 必须提供具体修复代码

---

## ✅ 5步流程

```
1. 扫描: ./scripts/quick-scan.sh
2. 验证: 用 Read 查看代码
3. 分类: P0/P1/P2
4. 修复: 提供具体代码
5. 验证: 重新扫描
```

---

## 📊 问题分级

| 级别 | 符号 | 修复时限 | 示例 |
|------|------|---------|------|
| 严重 | 🔴 P0 | 立即 | 崩溃、漏洞、数据丢失 |
| 中等 | 🟡 P1 | 尽快 | 类型安全、性能、空值检查 |
| 轻微 | 🟢 P2 | 有时间 | 命名、注释、风格 |

---

## 🎓 核心原则

1. **实际 > 理论** - 解决代码中实际存在的问题
2. **扫描 > 理解** - 先用工具扫描发现问题
3. **数据驱动 > 文档驱动** - 从扫描数据出发
4. **简单工具 > 复杂AI** - grep/ast 更可靠

---

## ⏱️ 时间估算

- 快速扫描: 30秒
- 验证问题: 2-5分钟/问题
- 提供修复: 5-10分钟/问题
- 验证修复: 5分钟/问题

**预计总时间**: 10-20分钟/10个问题
